2. call 和 apply 的区别，哪个性能更好一些？

   都是 Function 原型上的方法，都是用来改变 this 的指向，call 传参逐个传递， apply 传参以一个数组传递，改变this 指向的还有一个函数 bind，该函数只是预先处理 this 的指向，函数不立即执行。

   args.length >3 call 性能更好些，反之则性能差不多。Jquery 源码中作者有过注释

4. 箭头函数与普通函数的区别？构造函数可以使用 new 创造实例，普通函数可以吗？为什么？

   - 箭头函数属于ES6，书写简洁（ES6中每一种函数都可以使用形参赋默认值和剩余参数运算符）；
   - 箭头函数没有自已的 this，其属于所属上下文（使用call/apply都无法改变）
   - 箭头函数中没有arguments，只能基于形参…arguments获取剩于参数集
   - 箭头函数不能被 new 执行，箭头函数没有自已的 this，也没有 prototype 属性（也就没有 prototype.constructor）
